S2-002

漏洞原理

官方描述:

<s:url> 和 <s:a> 标签在生成 URL 时，如果包含的参数值没有被正确转义，攻击者就可以注入恶意的客户端脚本代码。

具体情况：

  • <s:a> 标签中的参数值，如果包含未转义的双引号 ", 可以破坏 HTML 属性，并插入恶意代码。
  • <s:url> 和 <s:a> 标签在 includeParams 不为 "none" 时，无法正确转义 <script> 标签。
    • 例如攻击 URL：

​ 这样会导致浏览器执行恶意脚本。

就是XSS漏洞，影响组件即Strust2<s:url>和 <s:a> 标签，允许攻击者注入 任意 JavaScript 代码，可能窃取用户 Cookie、篡改页面、实施钓鱼攻击。

漏洞影响版本

Struts 2.0.0 到 2.1.8.1

解决方案

  • 官方建议：立即升级 Struts2 至 2.2.1 或更高版本。

  • 原因：

    • 从 Struts 2.0.11.1 开始：
      • <s:a> 标签会对双引号进行转义。
      • <s:a> 和 <s:url> 标签会对 <script> 标签递归转义。
    • 但是后来发现 2.0.11.1 并不能完全修复问题，只有 2.2.1 及以上版本 才彻底解决
