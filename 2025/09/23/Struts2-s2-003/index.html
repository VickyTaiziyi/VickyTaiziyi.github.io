Struts2-s2-003

发表于 2025-09-23 分类于 Struts2 阅读次数：

S2-003

漏洞原理

官方描述：

Struts2 在处理请求参数时，默认会用 ParametersInterceptor 来把 URL 参数设置到对应的 Action 属性中。
为了避免被攻击，Struts2 给 # 的使用加了保护机制（因为 # 在 OGNL 中表示访问上下文对象，比如 #session、#application 等）。

但是攻击者发现可以通过 Unicode 编码 和字符串拼接来绕过限制，例如：

其中：

  • \u0023 就是 #
  • 拼接 'session\'user\'' 后变成 #session['user']
  • 这样一来，OGNL 就能修改服务端的上下文对象，比如会话里的用户数据。

也就是说，攻击者只要构造恶意请求参数，就可以 直接修改服务端 session、application 等对象的值，造成严重后果。

关键:攻击者不直接把 # 写在参数名里，而是利用 OGNL 的表达式能力（字符串拼接、unicode 转义等）构造会在 OGNL 内部解析后成为 #session.user 的表达式，但外观上并不包含字面 #

漏洞影响版本

Struts 2.0.0 - 2.1.8.1

解决方案

官方建议：

  • 立即升级 至 Struts 2.2.1 或以上版本。
  • 不要依赖黑名单/简单过滤 #，而是彻底升级框架，修复底层的 OGNL 处理逻辑。
